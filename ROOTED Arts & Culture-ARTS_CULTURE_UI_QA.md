# ROOTED Arts & Culture — UI Front-End QA Requirements

This vertical is cultural, educational, and trust-sensitive.

Backend authority ALWAYS wins.

---

## 1. Discovery UI

MUST:
- Display 6–8 venue cards max  
- Apply specialty filters (museum, gallery, theatre, history)  
- Respect backend rotation  
- Enforce geographic limits (≤ 50 miles)  

MUST NOT:
- Display ranking mechanisms  
- Suggest “popular” or “trending” content  

---

## 2. Venue Profiles

Show:
- Description  
- Hours  
- Gallery  
- Specialty  
- Accessibility info  

Hide:
- Analytics  
- Internal ratings  
- Messaging to minors  

---

## 3. Kids Mode

Remove:
- Mature exhibits  
- Political art  
- Religious art (unless unlocked)  
- Donation prompts  
- Ticket pricing  

---

## 4. Events UI

Must:
- Clearly show start/end times  
- Use clean family-friendly layouts  
- Distinguish workshops vs performances  

Must not:
- Blend adult and kids content  
- Show adult messaging tools  

---

## 5. Map UI

- Cultural markers only  
- Seasonal overlays allowed  
- Street-level exactness suppressed for Kids Mode  

---

END OF QA
